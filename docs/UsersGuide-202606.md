# GNSS Scope / NMEA 分析アプリ — 利用者の手引

- **文書バージョン**：2026.1
- **最終更新**：2026-06-10
- **対象アプリ**：GNSS Scope（`index.html`）／対応機能仕様書：`docs/funcspec-202606.md`

> 本アプリは、PicoW + u-blox MAX-M10S が WebSocket で流す生 NMEA を受信し、
> 測位の有効性・衛星の見え方・精度・データ品質をリアルタイムに可視化・収録するツールです。
> この手引は、はじめて使う方が接続から収録までを一通り行えることを目的としています。

---

## 1. はじめに

### 1.1 できること
- M10S の測位状況（Fix種別・使用衛星・DOP・座標）をその場で確認できます。
- 衛星の配置（スカイプロット）と信号強度（SNR）を見て、空の見え方を把握できます。
- 使用衛星数・HDOP・平均SNR の推移を時系列で追えます。
- 受信品質（チェックサム通過率・更新間隔のばらつき・欠損率）を点検できます。
- 受信した生データを収録し、あとで解析に使えます。

### 1.2 動作環境
- **ブラウザ**：Google Chrome / Microsoft Edge など Chromium 系を推奨。
- **配信**：ファイルを直接開く（`file://`）方式では動きません。HTTP で配信してください（→ 2章）。
- **ネットワーク**：
  - **ライブ受信は HTTP（localhost / 自宅LAN）で開いてください。**
    HTTPS のページからは `ws://` に接続できません（ブラウザの mixed content 制限）。
  - 収録データの解析・共有は HTTPS でも可能です（役割分担）。

## 2. 起動方法

ES モジュール構成のため、ローカルサーバ経由で開きます。アプリのフォルダ
（`index.html` がある場所）で次のいずれかを実行します。

```bash
# Python がある場合
python -m http.server 8000

# Node.js がある場合
npx serve .
```

ブラウザで `http://localhost:8000` を開くとアプリが表示されます。

## 3. 画面の構成

### 3.1 上部バー
左から、接続状態を示すドット、アプリ名、操作コントロールが並びます。

| 項目 | 説明 |
|---|---|
| WS URL 入力 | 接続先。既定は `ws://picow.local/`。必要に応じて IP/ポートに変更します。 |
| 接続 / 切断 | WebSocket の接続・切断を切り替えます。 |
| 収録開始 / 収録停止 | 受信中の生データの収録を開始・停止します。 |
| デモ再生 / デモ停止 | Pico が無くても動作確認できる合成データを流します。 |
| 接続状態 | `disconnected / connecting / connected / reconnecting / demo` を表示。 |
| 収録状態 | `—`（未収録）／`収録中…`／`停止（N 行）`。 |

接続状態ドットの色：緑＝接続中・デモ中／黄＝接続試行・再接続中／赤＝未接続。

### 3.2 本体（5つのパネル）
1. **測位ステータス**：Fix種別・測位モード・使用衛星・DOP・座標・標高・UTC・有効測位率。
2. **スカイプロット**：衛星の方位・仰角の配置図。
3. **SNR（C/N0）**：衛星ごとの信号強度の棒グラフ。
4. **時系列**：使用衛星数 / HDOP / 平均SNR の推移（直近300秒）。
5. **データ品質**：受信品質の指標と、エポック間隔の分布。

画面幅が 720px 以下になると、パネルは縦に積み重なって表示されます。

## 4. 基本的な使い方

### 4.1 ライブ受信する
1. Pico W（M10S）を起動し、WebSocket サーバが立ち上がっていることを確認します。
2. **HTTP** でアプリを開きます（2章）。
3. WS URL に接続先（例 `ws://picow.local/` または `ws://192.168.x.x/`）を入力します。
4. **接続** を押します。状態が `connected` になり、各パネルが更新され始めます。
5. 終了するときは **切断** を押します。

> 接続が切れると自動で再接続を試みます（最大15秒間隔まで徐々に延ばします）。
> `reconnecting` 表示のまま戻らない場合は、URL や Pico 側を確認してください。

### 4.2 デモで試す（Pico なし）
1. **デモ再生** を押すと、合成された NMEA が 1Hz で流れ、全パネルが動きます。
2. 各表示の見方を確認するのに使えます。**デモ停止** で止まります。
   （デモ中に「接続」を押すとデモは止まり、ライブ接続に切り替わります。）

### 4.3 収録する
1. 受信中（ライブまたはデモ）に **収録開始** を押します。状態が `収録中…` になります。
2. 受信した生の行がブラウザ内（IndexedDB）にセッション単位で保存されます。
3. **収録停止** を押すと `停止（N 行）` と保存行数が表示されます。

> 収録は接続・デモとは独立に開始・停止できます。
> 収録データはブラウザ内に保存され、同じ生データから後で同じ解析結果を再現できます。
> （保存セッションを選んで再生する画面UIは今後の対応予定です。）

## 5. 各パネルの見方

### 5.1 測位ステータス
- **Fix種別バッジ**（色分け）：
  - `No fix`（赤）＝測位なし
  - `GPS fix` / `DGPS`（標準色）＝単独測位 / ディファレンシャル
  - `RTK fixed` / `RTK float`（緑）＝高精度測位
  - `Dead reckoning`（黄）＝推測航法
- **測位モード**：`No fix` / `2D` / `3D`。
- **使用衛星 / HDOP / PDOP / VDOP**：測位に使った衛星数と精度劣化指標（小さいほど良い）。
- **緯度 / 経度 / 標高 / UTC**：現在の測位値。データが来ていない項目は「—」。
- **有効測位率**：これまでのエポックのうち有効だった割合。
  `quality>0 かつ RMC が V（無効）でない` ものを有効として数えます。

### 5.2 スカイプロット
- 中心が天頂（仰角90°）、外周が地平線（仰角0°）。**北が上・東が右**、時計回りに方位。
- **●（塗り）＝測位に使用中**、**○（中抜き）＝可視のみ**。
- **円の大きさ＝SNR**（大きいほど強い）。色はコンステレーション（凡例参照）。
- 各点の下に衛星番号（PRN）を表示します。

### 5.3 SNR（C/N0, dBHz）
- 可視衛星ごとに棒1本。コンステ順 → PRN順に並びます。
- **濃い棒＝使用中、薄い棒＝可視のみ。** 目盛りは 0 / 20 / 40 dBHz。
- SNR が取れていない衛星は表示されません。一般に 40 dBHz 前後あれば良好です。

### 5.4 時系列
- 直近 300 秒のスクロール窓で推移を表示します。
  - **左軸＝平均SNR[dBHz]**、**右軸＝使用衛星数**、**破線＝HDOP**。
- 横軸は受信開始からの経過秒です。樹林下や谷筋で値がどう荒れるかを追えます。

### 5.5 データ品質
- **チェックサム通過率**：正しく受信できた割合（高いほど良い）。
- **不正センテンス**：チェックサムに失敗した行数。
- **エポック数 / GGA欠損率**：処理したエポック数と、座標核である GGA を欠いた割合。
- **平均エポック間隔 / ジッタ(σ)**：更新間隔（1Hz なら理想は約1000ms）とそのばらつき。
- **センテンス更新レート**：種別（GGA/RMC/GSA/GSV）ごとの累計と Hz。
- **エポック間隔の分布**：間隔のヒストグラム。1000ms 近傍が強調され、外れ値が分かります。

## 7. 困ったときは

| 症状 | 確認すること |
|---|---|
| 接続できない / `reconnecting` のまま | URL（ホスト・ポート）と Pico 側のサーバ稼働を確認。ページを **HTTP** で開いているか確認（HTTPS からは `ws://` 不可）。同一LANにいるか確認。 |
| 画面が真っ白・パネルが動かない | `file://` で開いていないか確認（HTTP配信が必要）。Chromium 系ブラウザか確認。 |
| 時系列グラフが出ない | uPlot（CDN）を読み込めているか確認。オフライン時はネット接続またはキャッシュを確認。 |
| 値が「—」のまま | 該当センテンスが届いていない可能性（例：GGA未受信で座標が空）。データ品質の更新レートを確認。 |
| 収録した行数が 0 | 収録開始のタイミングで受信していたか確認（接続またはデモが動いているか）。 |
